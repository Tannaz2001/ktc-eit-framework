Sprint 5: HD95 metric, BackProjectionPlugin, real data wired end-to-end

HD95 metric (src/ktc_framework/metrics/ktc_score.py):
- hd95(pred, gt, label) using scipy distance_transform_edt — O(N) efficient
- Computes 95th-percentile Hausdorff Distance per class (resistive/conductive)
- Returns 0.0 when either mask is empty (no surface to compare)
- Registered as hd95_resistive and hd95_conductive in experiment_runner

BackProjectionPlugin (src/ktc_framework/methods/back_projection_plugin.py):
- Simple linear back-projection: voltages (2356,) → (256,256) segmentation
- Reshapes to (76,31), computes per-electrode sensitivity via injection-weighted
  voltage magnitudes, interpolates onto 256×256 grid with scipy griddata (cubic)
- Gaussian smoothing + threshold into {0=background, 1=resistive, 2=conductive}
- Registered as BackProjectionPlugin via @register decorator

Runner updates:
- Auto-imports BackProjectionPlugin on startup
- Summary table now shows HD95 Res. and HD95 Cond. columns
