# Baseline Strategy: No Flying Blind

**Problem**: We don't have published KTC 2023 baseline scores, so we can't tell if our methods are working.

**Solution**: Establish empirical baselines from the training data we have locally, then use those as ground truth.

---

## The Approach (3 Phases)

### Phase 1: Establish Baselines (LOCAL DATA)
- **Script**: `establish_baselines.py`
- **Data**: 4 training samples (stored in `Codes_Matlab/TrainingData/` + `Codes_Matlab/GroundTruths/`)
- **Process**:
  1. Load each training sample
  2. Run all methods (MockBaseline, BackProjection, GaussNewton)
  3. Compute KTC scores
  4. Generate `baselines.json` with per-sample scores
  5. Generate `BASELINE_REPORT.md` with statistics

**Output**: Ground truth baseline table showing what each method scores on training data

### Phase 2: Build Test Harness (USES BASELINES)
- **Script**: `test_harness.py` (to be built)
- **Purpose**: Run methods on evaluation data, compare to baselines
- **Logic**:
  ```
  For each evaluation sample:
    - Run method
    - Compute KTC score
    - Compare to baseline mean ± 1.96*std
    - Flag if significantly different
  ```
- **Output**: Pass/Fail report for each method at each level

### Phase 3: Analyze Degradation (VALIDATION)
- **Compare**: Training scores vs Evaluation scores
- **Expected**: 30-70% score degradation at Level 4 (due to 71% data loss)
- **Flag**: Methods that degrade > 90% or improve (impossible)

---

## Why This Works

### The Logic
```
Training data = known good (we can test locally)
              ↓
Establish baselines from training data
              ↓
Use baselines as reference
              ↓
Evaluation data = test on known reference
              ↓
Detect methods that break (vs methods that just struggle with sparse data)
```

### What We're Measuring
- **Training scores**: What the method CAN do (best case)
- **Evaluation scores**: What the method DOES do (real scenario)
- **Ratio**: How well it generalizes (should degrade gracefully, not collapse)

### Step 2: Build test_harness.py (AFTER baselines)
```python
# Pseudo-code for test_harness.py
import json

baselines = json.load(open("baselines.json"))

for level in [1, 2, 3, 4, 5, 6, 7]:
    for sample in ["A", "B", "C"]:
        for method in [BackProjection, GaussNewton]:
            score = run_method_on_sample(method, level, sample)
            baseline = baselines[method]["mean"]
            std = baselines[method]["std"]
            
            if abs(score - baseline) > 3 * std:
                print(f"FAIL: {method} L{level} deviates significantly")
            elif score < baseline - 0.8 * std:
                print(f"OK: {method} L{level} deviates as expected")
            else:
                print(f"OK: {method} L{level} normal")
```
